# Knovis prototype

Исполнимый прототип графа организационных знаний для поддержки крупного enterprise-продукта.

Проверяемый сценарий:

`роль -> задачи -> требования -> состояние сотрудника -> дефициты -> готовность -> минимальное действие развития`.

## Запуск

Рекомендуемый вариант на Windows: дважды нажать `open-knovis.cmd`.

Он запустит локальный сервер и откроет прототип в браузере.

Самый простой вариант: открыть `index.html` в браузере.

Локальный сервер:

```powershell
node server.js
```

После запуска открыть:

```text
http://127.0.0.1:5173
```

## Что уже есть

- демонстрационный домен enterprise support: L1, L2, Incident Manager, SRE, Customer Success, Knowledge Manager;
- правая навигация с разделами `Люди` и `Роли`;
- режим роли: выбор роли показывает только граф знаний роли;
- в граф добавлен слой `Фундамент`: он связывает задачи с более базовыми знаниями, без которых выполнение задачи ненадежно;
- часть задач теперь связана с несколькими фундаментами, например `Диагностика сложного обращения`, `Координация war room` и `Настройка наблюдаемости`;
- узлы `Роль`, `Задача` и `Фундамент` открывают описание и связанные сущности по клику;
- вершины графа можно перетаскивать, чтобы вручную уточнять раскладку;
- в режиме роли риски и действия развития скрыты из графа и открываются в карточке требования по клику на узел знания/навыка;
- режим человека: выбор человека открывает выбор роли и затем показывает оценку готовности;
- паспорт роли;
- граф связей роли, задач, требований, рисков и действий;
- оценка требований по сотруднику;
- расчет готовности;
- список дефицитов;
- минимально достаточные интервенции для закрытия дефицитов.
